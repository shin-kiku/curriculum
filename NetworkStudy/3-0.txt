ネットワーク層
・IP（Internet Protocol)
相手先のアドレス情報をもとにしてパケットをを送信する機能
インターネット層のプロトコル

・IPヘッダ
IPパケットのヘッダ分のこと
IPパケットの先頭にある、一定の形式にしたがって制御情報などを記載した部分のこと

・ICMP(Intenet Control Message Protocol)
IP通信の疎通状態を調査したり、正常に通信できない場合に送信先ホストへエラー情報を返したりする、
IP通信を補佐するためのプロトコル

・ICMPメッセージ
問合せやエラーに関するメッセージをやり取りすることでインテーネット通信において様々な役割を果たします
IP通信の判定、フロー制御、Ping要求・応答、時間経過、Redirect機能、アドレスマスク要求・おうとうなどの役割を
はたす

・IPアドレス
ネットワークに接続されたパソコンやプリンタなどに割り当てられる番号

・オクテット
IPv4において28ビットずつ、カンマで区切られた部分

・ネットワーク部
IPアドレスが所属しているネットワークを識別するための部分
そのIPアドレスが、どのネットワークで使われているのかがわかる

・ホスト部
ネットワーク内のコンピュータを識別するための部分

・クラス
オブジェクト指向プログラミングにおけるオブジェクトの雛形

・プライベートIPアドレス
LANなどの小規模ネットワークに存在する端末が、自由に使用できるIPアドレスのこと

・グローバルIPアドレス
インターネットに接続されているコンピュータや通信機器を個々に特定するために割り当てられた
IPアドレスのこと
インターネットに接続する際に必ず必要

・ネットワークアドレス
ネットワーク自体を指すアドレス

・ブロードキャストアドレス
ネットワーク内の全ての機器にデータを一斉配信するために使われる特殊なアドレス

・ループバックアドレス
ネットワークアドレスとして使用禁止とされているネットワークの「127.0.0.0/8」の
ネットワークでこのアドレスをループバックアドレスという
自分自身を示す仮想的なIPアドレス

・サブネット
ネットワーク内のネットワーク

・サブネットマスク
ネットワークの範囲（サブネット）を指定するために必要な番号

・プレフィックス表記
IPアドレスとサブネットマスクを一緒に表記する方法

・デフォルトゲートウェイ
通信機器などのネットワーク設置の一つで、送信相手までの経路がわからない場合に
とりあえずデータを送信する中継機器のIPアドレス

・IPv6(Internet Protocol Version 6)
各コンピュータに割り振られるIPアドレスを128ビットのデータとして表現するもの
最大約340澗台のコンピュータがインターネットに直接接続できる
128ビットとなるため2進数で128での表記となりますが、16ずつで区切り16進数で表すのが基本
IPv6はコロン区切りとなる

・サブネットプレフィックス

・インターフェイスID
IPv4におけるホスト部

・ルータ
コンピュータネットワークにおいて、データを２つ異常の異なるネットワーク間に中継する通信機器
ネットアーク層において、どのルートを通してデータを転送すべきかを判断するルート選択機能を持つ

・コンソール接続

・VTY接続

・AUX接続

・ルーティングテーブル
TCP/IPネットワークにおいてルータがパケットの転送先を決定するために記録しているネットワーク経路情報の一覧
宛先と次の転送先を一組にした経路をレキょしたもの

・ネクストホップ
次に転送する最も近いルータやインターフェイスのこと

・ARPテーブル
ARPによって割り出したIPアドレスとMACアドレスの対応表
・ARP（Address Resolution Protocol)
イーサネットのIPアドレスからMACアドレスを動的にしるためのアドレス解決プロトコルのこと
ネットワーク機器などがイーサネットフレームを送信する時に宛先のIPアドレスをもとにして、宛先のMACあdレスを調べるために使われる

・コアルータ
期間ネットワークにおいてメインで用いられるルータ

・センタールータ
企業で中心的に使用されるルータ
拠点となる本社や、データセンターなどの企業内で中心となる場所を接続するためのネットワークを構成する

・エッジルータ
企業で利用されるルータ
中心となる拠点を除く、営業所や支店などの拠点間を接続することを目的として利用されるルータ


・ブロードバンドルータ
家庭環境やSOHO環境など、インターネット回線に接続することを目的として利用されるルータ

・ルーティング
異なるネットワークにパケットを送信する際に最適な経路を決めること

・直接接続
・スタティックルーティング
管理者が宛先ネットワークへの最適なルートを手動で設定したルートのこと

・ダイナミックルーティング
ルーティングプロトコルにより動的に追加されるルートのこと

・ディスタンスベクタ型
ホップ数をメトリックとし、経路を選ぶ方式

・リンクステート型
格ルータが全体の接続情報を見て、最適経路を選択する方式

・ハイブリット型
ディスタンスベクタ型をベースに、リンクステート型を組み合わせた方式

・トランスポート層
通信プロトコルの機能や役割を階層構造で整理したモデルを構成する層
データの送信もとと送信先の間での制御や通知、交渉などを担うもの

・コネクション型通信
データ通信において、通信を開始する前に相手との間で仮想的な専用通信路（コネクション）を確立し、
それを通じてデータの送受信を行う通信方式
送信たデータが相手に確実に届き、データの到着順も送信順通りになることが保証される

・ACK
肯定応答/確認応答
二者間の通信で相手方に何らかの肯定的な応答を返す際に送られる信号やデータ、パケットなどのこと
「接続要求を受理した」「データが正しく受信できた」といった内容を伝達するために用いられる

・コネクションレス型通信
相手との接続（コネクション）確立を行わずデータを送信する方式
通信を始める際の事前処理や通信中の状態確認を行わないため通信のしんんらい生は高くありませんが、その分、通信コストが
低く、遅延が少ないという特徴を持ちます。

・TCP(Transmission Control Protocol)
１対１のセッションによる信頼性の高い通信を行うためのプロトコル
インターネットの主要なプロトコルの１つである。
パケットの再送やエラー訂正などを行う機能を持っているため、確実性をもった通信をしたい場面で利用される

・スリーウェイハンドシェイク
IPネットワーク上でTCPいよる接続を確立するための手順
パケットの送受信を３回行うからこのように呼ぶ

・Telnet
ネットワークを介して２つのホスト間で行うためのプロトコル
遠隔地にあるホストを仮想端末ソフトを利用して操作することができる
サーバ・クライアント方式で提供され、Talnetサーバが操作される側、クライアントが操作する側で動作する

・HTTP（Hyper Text Transfer　Protocol)
データをサーバとWebブラウザなどのクライアント間でやり取りする際の”通信手順”を意味する 

・FTP（File Transfer　Protocol）（ファイル転送プロトコル）
 パソコンから遠く離れた場所にあるパソコンにファイルを送ったり、逆に離れたパソコンからファイルをコピーしたり
するための仕組み

・SMTP（Simple Mail Transfer Protocol）
TCP/IPネットワークで標準的に用いられる、メールを送信するためん尾通信プロトコル

・TCPヘッダ
TCPパケットの先頭部分にある、制御情報などを一定の形式で記述した領域
この後ろに送りたいデータ本体（ペイロード）を加えて、一つのTCPパケットとなる

・UDP（User Datagram　Protocol） 
アプリケーション同士が最小限の仕組みででーたを送受信できるように考案されたシンプルなプロトコル

・RTP
TCP/IPネットワーク上で音声や動画のように連続するでーたの流れをリアルタイムに伝送するための通信プロトコル

・RTCP
RTPなどを用いてデータをリアルタイムで伝送する際に、送受信制御などを行うプロトコル

・DHCP（Dynamic Host Configuration Protocol)
IPv4ネットワークにおいて通信用の基本的な設定を自動的に行うためのプロトコル

・TFTP（Trivial File Transfer Protocol）
特定のコンピュータ間でファイル転送する時に使用するアプリケーション層プロトコル

・RIP
ルーティングプロトコルの一つ
ルータなどの通信ん機器の間で経路情報を交換し、ある地点から別の地点までの最短経路を割り出すのに使う
